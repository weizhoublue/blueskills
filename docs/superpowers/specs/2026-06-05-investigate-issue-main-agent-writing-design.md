## 背景

`investigate-issue` v0.4.0 已采用单 `SKILL.md` + Markdown 阶段流转。当前工作流中：

- **阶段 3**：主编排仅在上下文中综合阶段 1、2a、2b，不产出三节报告。
- **阶段 4**：再委派「报告撰写员」sub-agent，根据综合摘要写三节初稿。
- **阶段 5**：委派「报告评审员」；`needs_enrichment` 时委派「报告补充员」改稿。

阶段 3 与 4 之间多一次摘要传递与 Task 往返；而阶段 3 的综合工作本就在主编排上下文中完成，与 `plugins/audit/skills/review/SKILL.md` 中「分析委派、报告拼装在主线程」的模式可对齐。

**用户诉求（方案 A）**：仅合并阶段 3+4，初稿由主编排撰写；阶段 5 的**评审员**与**补充员**仍独立委派。

## 目标

1. 主编排在**阶段 3**内完成：内部分析综合 + 三节初稿（原阶段 3+4）。
2. **保留**原阶段 5（现**阶段 4**）的「报告评审员」与「报告补充员」sub-agent 委派不变。
3. 保留 R15–R20、C0–C4、B1–B5、W1–W3、`MAX_REVIEW_ROUNDS=3`、rollback 逻辑（rollback 后重做阶段 3，不再委派撰写员）。
4. 对外行为不变：终稿仍仅 **stdout** 输出，含 `REVIEW_RESULT=`。

## 非目标

- 不修改 `investigate-project`。
- 不改变阶段 1、2a、2b 的 sub-agent 委派与 Markdown 模板。
- 不将阶段 5 评审员或补充员并入主编排。
- 不新增磁盘中间产物或 JSON/jq。

## 方案选择

| 方案 | 描述 | 结论 |
|------|------|------|
| **A** | 仅合并阶段 3+4；评审+补充仍委派（现阶段 4） | **采用** |
| B | 合并 3+4；阶段 5 仅委派评审，改稿在主线程 | 拒绝 |
| C | 评审与改稿均在主线程 | 拒绝：自审盲区过大 |

## 新工作流

```text
阶段0：自检 + issue_brief
阶段1：问题信息搜集（sub-agent）→ ## 问题信息搜集结果
阶段2：并行
  2a 代码追踪（sub-agent）→ ## 代码追踪结果
  2b 业务上下文（sub-agent）→ ## 业务上下文分析结果
阶段3：主编排 — 综合 + 撰写三节初稿（不委派）    ← 原阶段 3+4
阶段4：整稿深化 ≤3 轮
  - 委派：报告评审员
  - needs_enrichment：委派报告补充员 → 完整三节
  - rollback：重跑 2a → 主编排重做阶段 3；round ← 1；最多 1 次
阶段5：主编排组装 stdout 终稿
```

**阶段编号**：连续为 0/1/2/3/4/5（合并原 3+4 后不再保留空缺的阶段 4 编号）；SKILL 内阶段 3 标题注明「含原综合分析与三节初稿」。

## 主编排：阶段 3

### 输入

- `issue_brief`
- 阶段 1 Markdown 全文
- 阶段 2a、2b Markdown 全文
- 全局规则全文

### 步骤

1. **内部分析综合**（编排上下文，可不单独对外输出）  
   合并：问题摘要与关键词、C0–C4 与缺陷落点、触发条件与后果、B1–B5 与兄弟对比、不触发场景、关键机制动机（若有）、未能确认的主张。

2. **撰写三节初稿**  
   直接按 SKILL 模板与 R16–R20 写出完整三节 Markdown（编排上下文保留，供阶段 4 评审/补充粘贴；不写磁盘文件）。

### 三节结构与规则

与原阶段 4 完全一致，包括但不限于：

- `## 1. 问题描述`：`### 业务上发生了什么` → `### 关键机制为何如此设计`（W1/W2/W3）→ `### 前因后果链` → `### 为何此处有问题、兄弟路径没有` → `### 代码佐证`（可选）
- `## 2. 触发条件`：正向须同时满足 → 故障表现 → 未能从代码确认的前提（若有）→ 不触发/正常情形 → 从输入到落点的过程
- `## 3. 结论`：整节仅一行 `REVIEW_RESULT=issue_true` 或 `REVIEW_RESULT=issue_false`

禁止 markdown 表格；专名/缩写首现须解释。

## 阶段 4：评审与补充（仍委派）

### 报告评审员

- **行为**：与 v0.4.0 一致（R16–R20 扫描、缺失清单、`complete` / `partial` / `needs_enrichment`）。
- **prompt 调整**：
  - 输入：当前三节全文 + `issue_brief` + 全局规则 + 阶段 1/2a/2b 关键块或阶段 3 分析要点（可选粘贴）。
  - 删除：「仅供阶段 4 撰写员使用的综合摘要」等已废弃表述。

### 报告补充员

- **行为**：与 v0.4.0 一致，无逻辑变更。
- **prompt 调整**：
  - 输入：当轮缺失清单 + 当前完整三节 + 全局规则。
  - 删除：对「报告撰写员」的交叉引用；可附阶段 1/2a/2b 或阶段 3 分析要点供补证据。

- 输出：**完整三节** Markdown（非增量）。
- 须逐条回应缺失清单；禁止与 `confirmed` 矛盾；禁止 markdown 表格。

### Rollback（最多 1 次）

条件不变：第 1 轮评审、`call_chain` 维度 `blocking` 条数 ≥ 2、且尚未 rollback。

```text
重委派代码追踪 sub-agent（附 suggested_addition）
主编排重做阶段 3（综合 + 三节初稿）
rollback_used ← true
round ← 1
continue
```

不再出现「重委派撰写 sub-agent」或「重执行阶段 3 合并后再委派阶段 4」。

## 从 SKILL.md 删除的角色

- **仅删除**「报告撰写员」（原阶段 4）角色块与委派步骤。
- **保留**「报告补充员」角色块与委派步骤（留在阶段 4 小节）。

## 全局规则：分工约束

**调整项：**

- 原「报告撰写 sub-agent 不得与 confirmed 主张矛盾」→ **主编排**撰写初稿时不得与 `confirmed` 主张矛盾。
- **保留**：报告补充 sub-agent 不得与 `confirmed` 矛盾；须逐条回应缺失清单。
- **保留**：报告评审 sub-agent 输出缺失清单后，报告补充 sub-agent 必须逐条补充（无法补充时说明「综合分析中暂无依据」）。
